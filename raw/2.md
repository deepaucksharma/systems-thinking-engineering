```mermaid
    Cap --> Review

    CoordLever -.->|increases| Cap
    
    Review -->|Pass| Test
    Review -->|Fail| WIP
    Test -->|Pass| Done
    Test -->|Fail| WIP
    
    Done -.->|Weak standards| Escape
    Escape --> Incident
    Incident --> Hotfix
    Hotfix -->|Re-enters queue| Backlog
```

**Little's Law annotation:**
```
Cycle Time = WIP / Throughput
To reduce cycle time:
  1. Reduce WIP (Protection lever)
  2. Increase Throughput (Skill + Coordination levers)
  3. Prevent rework (Standards lever)
```

## **Diagram 7: Control Loop with Time Delays**
*Shows manager intervention timing and oscillation risk*

```mermaid

        Work[Work Process]
        Output[Output:<br/>Shipped Software]
    end
    
    subgraph Feedback["📊 TELEMETRY (Sensor)"]
        Leading[Leading Indicators<br/>WIP, interrupts, pairing]
        Lagging[Lagging Indicators<br/>Cycle time, defects, throughput]
        Delay[⏱️ Feedback Delay<br/>1-4 weeks]
    end
    
    Act -->|Changes| Conditions
    Conditions --> Work
    Work --> Output
    Output --> Leading
    Leading --> Delay
    Delay --> Lagging
    Lagging --> Observe
    Leading -.->|Early signal| Observe
    
    Observe --> Diagnose
    Diagnose --> Decide
    Decide --> Act
    
    subgraph Timing["⏰ TIMING RULES"]
        T1[Give intervention 2 full cycles<br/>before judging]
        T2[Review cadence from Table 3:<br/>Crisis: 2x/week<br/>Stabilizing: 1x/week<br/>Functional: Biweekly<br/>Thriving: Monthly]
        T3[If leading indicators move<br/>but lagging don't: HOLD COURSE]
        T4[If nothing moves in 2 cycles:<br/>RE-DIAGNOSE]
        Oscillation[⚠️ OSCILLATION RISK:<br/>If you change strategy<br/>faster than feedback delay,<br/>you are the instability source]
    end
    
    Decide -.->|Check timing| T1
    Observe -.->|Set cadence| T2
    Delay -.->|Respect latency| Oscillation
    
    style Oscillation fill:#ff6b6b,color:#fff
    style Delay fill:#feca57
```

**Key timing parameters:**
- **Feedback latency**: 1-4 weeks depending on batch size and deploy frequency
- **Minimum intervention period**: 2 review cycles (don't thrash)
- **Leading vs lagging**: Leading indicators move first (1-2 weeks), lagging follow (3-6 weeks)
- **Nyquist limit**: Don't intervene faster than 0.5 × feedback delay


```
This is why "a zero anywhere beats strength everywhere else."
```

## **Diagram 11: Lever Access Decision Tree**

```mermaid
    ProtBlocked --> ProtFix2[Option 2: Force tradeoff decision<br/>What should slip?]
    
    Blocked -->|Development| DevBlocked[DEVELOPMENT BLOCKED:<br/>No budget for hiring/training]

    Blocked -->|Structure| StructBlocked[STRUCTURE BLOCKED:<br/>Cross-team boundaries]
    StructBlocked --> StructFix1[Option 1: Fix intra-team first<br/>Build credibility]
    StructBlocked --> StructFix2[Option 2: Propose map with data<br/>not just complaints]
```

**Key principle:**
Every lever has a **free or low-cost substitute** you can use while building the case for full access:
- Direction → Reframe ambiguity as options
- Protection → Make cost visible
- Development → Pairing before hiring
- Structure → Fix local before global
- Standards → Voluntary adoption first
- Feedback Speed → Shrink batches before buying tools
