# Literature Map: What Each Source Contributes to Which Block

This page maps the intellectual foundations to the specific blocks and variables they inform.

## Block E — Execution Physics

| Source | Contribution |
|--------|-------------|
| Reinertsen — *Principles of Product Development Flow* | Queuing theory, batch size economics, WIP limits, utilization curve, Little's Law application |
| Goldratt — *The Goal* | Theory of Constraints; fix the bottleneck first |
| Hopp & Spearman — *Factory Physics* | M[X]/M/1/∞ model; utilization-cycle time nonlinearity |
| Taleb — *The Black Swan* | Heavy-tailed distributions; why averages are misleading; tail risk |
| Kim et al. — *The Phoenix Project* | Three Ways; flow, feedback, experimentation in practice |
| Forsgren et al. — *Accelerate* | DORA metrics empirical foundation; four key metrics |

## Block P — People System

| Source | Contribution |
|--------|-------------|
| Edmondson — *The Fearless Organization* | Psychological safety definition, measurement, and organizational impact |
| Deci & Ryan — *Self-Determination Theory* | Autonomy, competence, relatedness as intrinsic motivation drivers |
| Maslach — *Burnout research* | Job Demands-Resources model; burnout as systemic collapse |
| Sweller — *Cognitive Load Theory* | Intrinsic, extraneous, germane load decomposition |
| Schein — *Organizational Culture and Leadership* | Three-level culture model; underlying assumptions as control layer |
| Kahneman — *Thinking, Fast and Slow* | Cognitive limitations; System 1/2 thinking; decision quality under load |
| Google Project Aristotle | Empirical confirmation of psychological safety as primary team effectiveness predictor |
| Forsgren et al. — *SPACE framework* | Multidimensional developer productivity measurement |

## Block A — Org Alignment & Structure

| Source | Contribution |
|--------|-------------|
| Skelton & Pais — *Team Topologies* | Four team types; three interaction modes; Conway alignment |
| Conway — *How Do Committees Invent?* | Conway's Law; org structure determines system architecture |
| March & Simon — *Organizations* | Bounded rationality; political behavior; local optimization |
| Pfeffer — *Power in Organizations* | Power asymmetry; resource dependency; political economy |
| Hurwicz, Maskin — Mechanism Design theory | Incentive compatibility; designing systems where honesty is rational |
| Williamson — Transaction Cost Economics | Why org boundaries exist; make-vs-buy; governance |
| Beer — *The Viable System Model* | Requisite variety; Ashby's Law; control hierarchy |

## Block V — Strategic Layer

| Source | Contribution |
|--------|-------------|
| Snowden — Cynefin framework | Domain diagnosis; appropriate constraint types by domain |
| Ries — *The Lean Startup* | Build-measure-learn; validated learning; pivot vs persevere |
| Christensen — *The Innovator's Dilemma* | Disruption dynamics; why good execution on wrong strategy fails |
| Rumelt — *Good Strategy Bad Strategy* | What strategy actually is; diagnosis-guiding-policy-coherent action |
| Gothelf & Seiden — *Lean UX* | Outcome over output; continuous discovery |

| Source | Contribution |
|--------|-------------|
